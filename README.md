# AI-Powered Document & Multimedia Q&A 🤖🎓

A full-stack web application designed for students, researchers, and professionals to seamlessly extract, query, and chat with insights from PDF documents, audio files, and videos.

Built using **FastAPI**, **React**, **Google Gemini GenAI**, **FAISS (Vector Search)**, **Whisper**, **Redis**, and **Docker**.

---

## 🌟 Key Features

*   **Multi-modal File Uploads:** Supports PDF (`.pdf`), Audio (`.mp3`, `.wav`), and Video (`.mp4`, `.webm`).
*   **Semantic Q&A (Vector Search):** Uses FAISS local indexes backed by Gemini embeddings to search exact paragraphs related to your question.
*   **Real-time Chat Streaming:** AI responses stream token-by-token directly into the UI.
*   **Click-to-Play Timestamps:** When you ask questions about audio/video, the chatbot returns exact timestamps. Click the timestamp button to auto-seek the internal media player to that exact moment!
*   **AI Auto-Summarization:** Automatically generates comprehensive summaries of long documents/videos upon successful processing.
*   **Multi-User Authentication:** JWT-based secure authentication system for private environments.
*   **Rate-limiting & Caching:** Redis-backed sliding-window rate limiting on all endpoints, and 10-minute caching on AI summaries to prevent LLM quota exhaustion.

---

## 🛠️ Architecture

*   **Backend:** Python 3.13 / FastAPI
*   **Frontend:** React / Vite / Lucide Icons (Styling: Custom Glassmorphism CSS)
*   **Database:** MongoDB Atlas (Metadata + Chunks)
*   **Vector DB:** FAISS CPU (Stored locally or on shared docker volume)
*   **AI Models:**
    *   **LLM / Embeddings:** Google `gemini-2.5-flash` and `text-embedding-004`.
    *   **Transcription:** OpenAI `whisper` (Local CPU transcription for videos and audio).
*   **Infrastructure:** Docker Compose (Nginx proxy, Redis container).

## 🚀 Setup & Running Instructions

### Prerequisites
*   Docker & Docker Compose installed.
*   A Google Gemini API Key.
*   A MongoDB Atlas connection URL (or local MongoDB).

### 1. Environment Variables Configuration
Duplicate the `.env.example` in the backend (if any) or create `./backend/.env`:

```env
GEMINI_API_KEY=your_gemini_api_key_here
MONGODB_URL=mongodb+srv://<user>:<password>@cluster.mongodb.net/?retryWrites=true&w=majority
DB_NAME=ai_qa_db
UPLOAD_DIR=uploads
FAISS_DIR=faiss_indexes

# JWT Auth
JWT_SECRET_KEY=change-me-to-something-secure
JWT_ALGORITHM=HS256
JWT_EXPIRE_MINUTES=1440

# Redis Config (Points to Docker container by default)
REDIS_URL=redis://redis:6379/0
RATE_LIMIT_REQUESTS=30
RATE_LIMIT_WINDOW_SECONDS=60
```

### 2. Run with Docker Compose (Recommended)
This command builds the Backend, Frontend, and Nginx proxy, and wires them to Redis.
```bash
docker compose up --build
```
*   **Frontend UI:** `http://localhost:80`
*   **Backend API Docs (Swagger):** `http://localhost:8000/docs`

### 3. Run Manually (Local Development)

**Start Backend:**
```bash
cd backend
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt
uvicorn app.main:app --reload
```

**Start Frontend:**
```bash
cd frontend
npm install
npm run dev
```

## 🔌 API Documentation (Summary)

The full interactive API documentation is automatically generated by FastAPI. Once the application is running, open your browser to:
👉 **[http://localhost:8000/docs](http://localhost:8000/docs)**

*   **POST** `/api/auth/register` - Create a new account.
*   **POST** `/api/auth/login` - Returns JWT access token.
*   **POST** `/api/upload` - Upload a media/PDF file. Processing starts in the background. (Auth Required)
*   **GET** `/api/status/{file_id}` - Poll for `ready` or `error` status.
*   **POST** `/api/ask-stream` - Ask a question. Returns a Server-Sent Events (SSE) stream of tokens followed by a final JSON metadata block containing sources and media timestamps. (Auth Required)
*   **POST** `/api/summary` - Fetch an AI summary of a processed file. Results are cached in Redis. (Auth Required)
